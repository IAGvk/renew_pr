Below is a **clear, source‑grounded breakdown** of *every* data stream required for 24/7 CFE analytics and whether it is easy vs. hard to fetch — with citations from the earlier research pulls.

# ✅ TL;DR (Short Answer)

### **You can build the first 80% of the analytics product using data that already exists in digital form for almost every enterprise. No new hardware, sensors, or physical metering equipment is required.**

Enterprises already have:

*   Utility bills / smart meter hourly data
*   PPA production data
*   Renewable certificate purchase records
*   Region-level hourly emissions factors
*   Access to GC (Granular Certificate) issuers that provide ready-to-ingest certificate attributes

The only time sensors/hardware would be needed is:

*   If the customer *does not* have hourly consumption data and wants sub-hour granularity (rare)
*   If a customer wants real-time device-level control or on-site DER optimization (future product, not required for CFE analytics)

***

# ✅ Full Breakdown: Each Data Source & Its Feasibility

The 24/7 CFE Copilot needs only **five** types of data:

1.  **Hourly electricity consumption**
2.  **Hourly renewable generation (onsite or PPA)**
3.  **Hourly emissions factors**
4.  **Certificate / EAC / GC data**
5.  **Market metadata (locations, timestamps, matching rules)**

Let’s go one by one.

***

# 1) **Hourly Consumption Data (Load Data)**

### **Does this require hardware?**

**No.** Almost all large enterprises already have this via:

*   Utility smart meter portals
*   Grid operator APIs
*   Building management systems (BMS)
*   Data center energy logs
*   Monthly bill with “interval data download”

### **Why this is feasible**

Every regulated grid in the world uses interval meters for commercial customers.

### **No new sensors needed.**

You simply need authentication to:

*   Utility web portals
*   Energy data brokers
*   Internal energy management system exports

**Therefore: this is digitally available today for >95% of commercial customers.**

***

# 2) **Hourly Renewable Production Data (from PPAs or Onsite Assets)**

### **Does this require hardware?**

**No.**

Solar/wind PPAs and on‑site systems already provide:

*   SCADA feeds
*   Inverter logs
*   OEM portals (e.g., SolarEdge, Enphase, Vestas)

### **Why available**

Even NREL’s O\&M studies show large PV and wind sites already have data-rich monitoring and standards for performance logs. [\[docs.nrel.gov\]](https://docs.nrel.gov/docs/fy24osti/85819.pdf), [\[docs.nrel.gov\]](https://docs.nrel.gov/docs/fy23osti/84457.pdf)

This data is:

*   Already collected
*   Already timestamped
*   Usually available via API or SFTP dumps

***

# 3) **Hourly Grid Emissions Factors (Carbon Intensity)**

### **Does this require hardware?**

**Absolutely not.** These datasets are fully public or available via API.

Examples include:

*   Electricity Maps APIs
*   EPA / EIA region-level grid data
*   European ENTSO-E emission factors

### **Why available**

EnergyTag & Google’s 24/7 matching efforts rely heavily on publicly accessible hourly emissions datasets. Google integrates national hubs, utility systems, and grid operator data to automate hourly tracking. [\[energytag.org\]](https://energytag.org/projects/matching-for-googles-24-7-goal/), [\[datacenter...namics.com\]](https://www.datacenterdynamics.com/en/news/google-to-expand-247-clean-energy-matching-with-flexidao/)

No customer-side hardware needed.

***

# 4) **Energy Certificates / Granular Certificates (EAC / REC / GC Data)**

### **Does this require hardware?**

**No** — this is registry data, not physical device data.

### Availability:

*   **FlexiDAO** can issue hourly Granular Certificates in 47+ countries using existing monthly certificates + hourly production data. [\[energytag.org\]](https://energytag.org/energytag-accredits-first-granular-certificate-issuers-marking-a-major-milestone-for-hourly-clean-energy-tracking/)
*   **Energinet** issues GCs directly for Denmark. [\[energytag.org\]](https://energytag.org/energytag-accredits-first-granular-certificate-issuers-marking-a-major-milestone-for-hourly-clean-energy-tracking/)
*   **Granular Energy** has global coverage for time-stamped GCs beyond Europe. [\[granular-energy.com\]](https://www.granular-energy.com/insights/global-energytag-accreditation?category=news)

### Why this matters

These issuers already treat hourly matching as a digital records problem — not a hardware/sensor problem.

GCs come as:

*   JSON / API objects
*   CSV registries
*   Blockchain-backed identifiers (FlexiDAO)

You can ingest certificates without needing *any* physical infrastructure.

***

# 5) **Market Metadata (locations, grids, matching rules)**

### **Does this require hardware?**

**No.**

This includes:

*   ISO bidding zones
*   Grid region identifiers
*   Time zones
*   Allowable matching rules

EnergyTag’s open standard documents these rules. The customer doesn’t need to install anything. All logic lives in software. [\[energytag.org\]](https://energytag.org/energytag-accredits-first-granular-certificate-issuers-marking-a-major-milestone-for-hourly-clean-energy-tracking/)

***

# ⭐ So… do we ever need sensors or meters?

### **Not for the analytics dashboard.**

You can deliver:

*   Hourly consumption vs. supply matching
*   Hourly emissions matching
*   Hourly GC matching
*   CFE% per region, site, or hour
*   Optimization recommendations

**100% with existing digital data feeds.**

### **Sensors/hardware only needed if (future scenario):**

1.  The customer wants **real-time DER coordination** (dispatch batteries, shift load, control EV chargers)
2.  You want **sub-15-minute granularity**
3.  You’re measuring **behind-the-meter assets with no monitoring system** (rare for large enterprises)

But these are *Phase 2 or 3 products*, not necessary for the core 24/7 CFE analytics MVP.
